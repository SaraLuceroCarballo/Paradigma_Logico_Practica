Módulo 5

Recursividad en Prolog

Cómo definir predicados que se refieren a sí mismos y las herramientas integradas que lo aprovechan.

Concepto ¿Qué es la recursividad?

La recursividad es que un elemento de software se defina en función de sí mismo. En el paradigma lógico, ese elemento son los predicados: una cláusula que en su cuerpo hace referencia al mismo predicado que está definiendo.

Clave conceptual En Prolog no hay variables de estado ni bucles. La forma natural de expresar procesos que se repiten o que requieren "profundidad" es la recursividad.

Para que una definición recursiva sea correcta necesita siempre dos partes:

Caso base
La cláusula que corta la recursión. No hace referencia al propio predicado. Define el caso más simple/mínimo del problema.
Caso recursivo
La cláusula que sí se refiere al propio predicado, pero con argumentos más pequeños o más cercanos al caso base. Sin esto la recursión sería infinita.
Error frecuente Definir solo el caso recursivo (sin caso base) provoca una pila de llamadas infinita que termina en un error de stack overflow.

Ejemplo Ancestros — recursión sobre una jerarquía

Supongamos que queremos saber quiénes son los ancestros de una persona. Declarativamente:

  • Mis ancestros son mis padres y
  • los ancestros de mis padres.

El y en castellano funciona como o lógico: alcanza con que se cumpla cualquiera de las dos condiciones. Por eso usamos dos cláusulas separadas:

Prolog · ancestros.pl
% Base de hechos
padre(tatara, bisa).
padre(bisa, abu).
padre(abu, padre).
padre(padre, hijo).

% Caso base: el padre directo es ancestro
ancestro(Padre, Persona) :- padre(Padre, Persona).

% Caso recursivo: el ancestro del padre también es mi ancestro
ancestro(Ancestro, Persona) :-
    padre(Padre, Persona),
    ancestro(Ancestro, Padre).

Consulta de ejemplo y resultados:

Consola SWI-Prolog
?- ancestro(tatara, Quien).
Quien = bisa ;
Quien = abu  ;
Quien = padre ;
Quien = hijo  ;
false.

Prolog recorre las cláusulas y encuentra todas las soluciones posibles.

¿Es inversible ancestro/2? Dejar ambas variables libres (?- ancestro(Ancestro, Quien).) sí es posible y devuelve todos los pares (ancestro, descendiente) de la base de conocimiento, siempre y cuando no haya ciclos. La inversibilidad es total.

Ejemplo Distancias entre ciudades — recursión con aritmética

Queremos calcular la distancia entre dos puntos que pueden estar conectados directamente o mediante escala. La base de hechos es:

Prolog · distancias.pl
distancia(buenosAires, puertoMadryn, 1300).
distancia(puertoMadryn, puertoDeseado, 732).
distancia(puertoDeseado, rioGallegos, 736).
distancia(puertoDeseado, calafate, 979).
distancia(rioGallegos, calafate, 304).
distancia(calafate, chalten, 213).

% Caso base: conocemos la distancia directa
kilometrosViaje(Origen, Destino, Kms) :-
    distancia(Origen, Destino, Kms).

% Caso recursivo: pasamos por un punto intermedio
kilometrosViaje(Origen, Destino, KmsTotales) :-
    distancia(Origen, PuntoIntermedio, KmsIntermedios),
    kilometrosViaje(PuntoIntermedio, Destino, KmsFinales),
    KmsTotales is KmsIntermedios + KmsFinales.

Este predicado encuentra todas las rutas posibles entre dos ciudades, no solo la más corta. Por eso para Puerto Madryn → Calafate existen dos respuestas: 1711 y 1772 km.

Problema: dirección única Si los hechos de distancia/3 solo están en un sentido, kilometrosViaje(calafate, buenosAires, Kms) devuelve false. Solución: agregar un predicado totalViaje/3 que admita ambas direcciones.
Prolog · viaje bidireccional
totalViaje(Origen, Destino, Kms) :-
    kilometrosViaje(Origen, Destino, Kms).
totalViaje(Origen, Destino, Kms) :-
    kilometrosViaje(Destino, Origen, Kms).

Ahora ?- totalViaje(calafate, buenosAires, Kms). devuelve 3011 y 3072 (las dos rutas posibles en ese sentido).

Predicado member/2 — pertenencia a una lista

Un elemento X pertenece a una lista si:

  • es la cabeza de esa lista, o
  • pertenece a la cola de esa lista (y aquí viene la recursión, porque la cola también es una lista).
Definición de member/2
% Caso base: X está en la cabeza
member(X, [X|_]).

% Caso recursivo: X está en la cola Z
member(X, [_|Z]) :- member(X, Z).

¿Qué pasa con la lista vacía? No hay caso para [] porque [] no tiene cabeza ni cola — es indivisible. Esto se conecta con el Principio de Universo Cerrado: lo que no puede probarse es falso, y ningún elemento puede pertenecer a una lista vacía.

Consultas de ejemplo
?- member(5, [5, 8]).    % true — 5 es la cabeza
?- member(5, [4, 6]).    % false — no está
?- member(5, [4, 5]).    % true — 5 está en la cola
?- member(5, [4, 3, 5, 6]). % true — profundidad 2

Inversibilidad de member/2. Como el primer argumento puede quedar libre, es posible consultar:

member/2 invertido
?- member(X, [3, 4, 6]).
X = 3 ;
X = 4 ;
X = 6.
¿Y si dejamos libre la Lista? ?- member(6, Lista). es técnicamente posible pero genera listas infinitas: [6|_], [_,6|_], etc. No tiene sentido práctico porque el universo de listas que contienen un 6 es infinito.

Predicado nth1/3 — elemento en posición N

nth1/3 relaciona una posición (contando desde 1), una lista y el elemento que ocupa esa posición. Es inversible: puede encontrar el elemento dada la posición o la posición dado el elemento.

Consultas de nth1/3
?- nth1(2, [1, 2, 3, 5, 8, 13], Elemento).
Elemento = 2.

?- nth1(Posicion, [1, 2, 3, 5, 8, 13], 5).
Posicion = 4.

Podemos implementar nuestra versión propia llamada enesimo/3 para entender cómo funciona internamente:

Implementación manual de enesimo/3
% Caso base: el elemento está en la cabeza → posición 1
enesimo(1, [Elemento|_], Elemento).

% Caso recursivo: buscar en la cola e incrementar posición
enesimo(Posicion, [_|Resto], Elemento) :-
    enesimo(PosicionCola, Resto, Elemento),
    Posicion is PosicionCola + 1.

La recursión avanza consumiendo la cabeza de la lista. Cuando el elemento está en la posición 1 de algún sufijo, el resultado se construye de vuelta sumando 1 por cada nivel al volver.

Resumen · Módulo 5

  • La recursividad en Prolog = un predicado definido en términos de sí mismo.
  • Siempre necesita caso base (sin auto-referencia) + caso recursivo.
  • Prolog puede encontrar múltiples soluciones para predicados recursivos si no hay restricciones de inversibilidad.
  • member/2 es inversible en el primer argumento; no tiene sentido dejar libre el segundo.
  • nth1/3 relaciona posición, lista y elemento; es inversible en los tres.
Módulo 6

Diseño: Delegación, Acoplamiento y Code Smells

Cómo estructurar predicados, medir su calidad y detectar errores conceptuales frecuentes.

Diseño Elementos de diseño y modelado

Diseñar es tomar tres decisiones:

  1. ¿Cuáles son los componentes?
  2. ¿Qué responsabilidades tiene cada uno?
  3. ¿Qué relaciones hay entre ellos?

Dentro del paradigma lógico los componentes son predicados e individuos. La responsabilidad de un predicado es la relación que define: member/2 determina pertenencia a una lista; forall/2 verifica que todos satisfagan una condición.

Caso práctico: empresa de remises

El relevamiento pide saber: qué choferes viajaron a Villa del Parque, quiénes manejaron cierto auto en cierta fecha, y qué viajes hizo un cliente. Identificamos las entidades:

Clientes
Se representan como functores: cliente(3, sonia).
Choferes
Átomos simples: matias.
Autos
Functores: auto(gol, "ZFE447").
Fechas
Functores: fecha(11, 2, 2017).
Viajes
Predicado de aridad 8 que agrupa todo lo anterior.
Prolog · modelado de viajes
viaje(cliente(3, sonia), fecha(11, 2, 2017),
     auto(gol, "ZFE447"), matias,
     "Dorrego y Roque Pérez", "Hudson", "Cadorna 283", "Wilde").
Functor vs. Predicado cliente/2 en el ejemplo es un functor (estructura de datos dentro de un argumento), no un predicado de primer nivel. Por eso el universo de clientes no es explícito — se deriva con un predicado generador: clienteReal(C) :- viaje(C, _, _, _, _, _, _, _).

Diseño Acoplamiento

El acoplamiento es el grado en que los componentes se conocen entre sí. No es inherentemente malo — es necesario para resolver requerimientos — pero conviene mantenerlo localizado.

Ejemplo: terminoAnio/2
terminoAnio(Alumno, Anio) :-
    forall(materia(Materia, Anio), aprobo(Alumno, Materia)).

aprobo(Alumno, Materia) :-
    nota(Alumno, Materia, Nota), Nota >= 6.

terminoAnio/2 conoce a materia/2 y a aprobo/2. Ese acoplamiento es buscado. A su vez, aprobo/2 conoce a nota/3. El árbol de dependencias es:

terminoAnio/2 → forall/2 → materia/1
                                ↘ aprobo/2 → nota/3

Si cambia el modelo de notas (de un escalar a una lista con aplazos), solo hay que tocar aprobo/2. El predicado terminoAnio/2 no necesita cambiar porque no conoce los detalles de cómo se guarda la nota:

❌ Modelo viejo (nota escalar)
nota(raul, pdp, 9).
nota(herminio, pdp, 6).

aprobo(A, M) :-
    nota(A, M, Nota),
    Nota >= 6.
✅ Modelo nuevo (lista de notas)
nota(raul, pdp, [9]).
nota(herminio, pdp, [2, 2, 6]).

aprobo(A, M) :-
    nota(A, M, Notas),
    last(Notas, Nota),
    Nota >= 6.

Solo cambió aprobo/2. terminoAnio/2 quedó intacto. Esto demuestra que el acoplamiento estaba bien diseñado: los detalles de representación están encapsulados detrás de un predicado de interfaz.

Code Smells Antipatrones en Prolog

Los code smells son señales de que la solución, aunque funcione, debería mejorarse. El proceso de mejora sin cambiar el comportamiento se llama refactorización.

A continuación los 10 code smells del módulo, con ejemplos concretos de código malo y su versión mejorada.

① Negación vs. Aserción

Usar doble negación cuando forall/2 expresa la misma idea de forma directa.

❌ Doble negación (difícil de leer)
todosSiguenA(Rey) :-
    personaje(Rey),
    not((personaje(P),
         not(sigueA(P, Rey)))).
✅ Aserción con forall
todosSiguenA(Rey) :-
    personaje(Rey),
    forall(personaje(P),
           sigueA(P, Rey)).

La transformación lógica es: ¬∃x / p(x) ∧ ¬q(x) equivale a ∀x / p(x) ⟹ q(x).

② Evitar duplicidades

No repetir lógica similar para distintos tipos de entidades. Unificar bajo un predicado polimórfico.

❌ Predicados separados para bares y museos
baresCopados(Ciudad, Bares) :-
    findall(bar(C),
      (puntoDeInteres(bar(C), Ciudad),
       C > 4), Bares).
museosCopados(Ciudad, Museos) :-
    findall(museo(cienciasNat),
      puntoDeInteres(museo(cienciasNat),
        Ciudad), Museos).
✅ Predicado unificado polimórfico
cosasCopadas(Ciudad, Cosas) :-
    findall(Cosa,
      (puntoDeInteres(Cosa, Ciudad),
       copada(Cosa)),
      Cosas).

copada(museo(cienciasNaturales)).
copada(bar(V)) :- V > 4.

Al agregar un nuevo tipo de lugar copado (ej: centro cultural), solo se agrega un hecho/regla de copada/1. El resto no cambia.

③ Keep it simple — existencia simple

Si solo necesitás saber que existe algo, no armes una lista para luego chequear su longitud.

❌ findall + length para existencia
inFraganti(Delito, Del) :-
    cometio(Delito, Del),
    findall(T, testigo(Delito,T), Ts),
    length(Ts, N),
    N > 0.
✅ Existencia directa con variable anónima
inFraganti(Delito, Del) :-
    cometio(Delito, Del),
    testigo(Delito, _).

Lo mismo aplica para verificar ausencia: en vez de findall + length(..., 0), usar not(padre(Persona, _)).

④ Keep it simple — hijo único

Verificar que alguien sea hijo único no requiere contar con una lista.

❌ Contar la lista de hijos
hijoUnico(Persona) :-
    padre(Padre, Persona),
    findall(H,padre(Padre,H),Hs),
    length(Hs, 1).
✅ No existe otro hijo distinto
hijoUnico(Persona) :-
    padre(Padre, Persona),
    not((padre(Padre, P2),
         Persona \= P2)).

⑤ findall + member — lista innecesaria

Si armás una lista con findall y enseguida la recorrés con member, las dos operaciones se cancelan. Es un anti-patrón clásico.

❌ findall luego member (lista inútil)
estaComplicado(P) :-
    findall(H,
      (padre(P,H),
       edad(H,E), E<18), Hs),
    member(Hijo, Hs),
    problematico(Hijo).
✅ Consulta directa
estaComplicado(P) :-
    padre(P, Hijo),
    edad(Hijo, Edad),
    Edad < 18,
    problematico(Hijo).

⑥ findall + forall(member, …) — lista innecesaria

Armar una lista para después iterar sobre ella con forall es redundante: directamente se puede usar el predicado generador dentro del forall.

❌ Generar lista y luego forall
feliz(G) :-
    gobernante(G),
    findall(D,diario(D),Ds),
    forall(member(D,Ds),
           oficialista(D,G)).
✅ forall directo sobre el generador
feliz(G) :-
    gobernante(G),
    forall(diario(D),
           oficialista(D,G)).
Excepción legítima forall(member(X, A), member(X, B)) es correcto dentro de incluido/2 porque el dominio ya es una lista — no la estamos generando.

⑦ Lazy predicate (predicado sin valor)

Un predicado que solo llama a otro con los mismos argumentos no agrega valor.

❌ Delegación vacía
estaComplicado(P) :-
    personaComplicada(P).

personaComplicada(P) :-
    % lógica real acá…
✅ Predicado único con responsabilidad propia
estaComplicado(P) :-
    padre(P, Hijo),
    edad(Hijo, E), E < 18,
    problematico(Hijo).

⑧ Inversibilidad faltante

Un predicado con forall que no tiene predicado generador no es inversible.

❌ Sin generador (no inversible)
viejoMaestro(P) :-
    forall(pensamiento(P,X),
           llegaANuestrosDias(X)).
✅ Con generador (inversible)
viejoMaestro(P) :-
    pensador(P),
    forall(pensamiento(P,X),
           llegaANuestrosDias(X)).

⑨ Incógnitas innecesarias

No uses una variable si el valor ya es conocido y constante.

❌ Variable local innecesaria
numeroDeLaSuerte(joaquin, N) :-
    N is 8.
✅ Hecho directo
numeroDeLaSuerte(joaquin, 8).

⑩ Expresividad y parametrización

Nombres poco descriptivos y predicados con condiciones hardcodeadas dificultan la lectura y el mantenimiento.

❌ Condiciones explícitas y hardcoded
puedeComer(analia, Comida) :-
    ingrediente(Comida, _),
    forall(ingrediente(Comida, I),
      (not(contieneCarne(I)),
       not(contieneHuevo(I)),
       not(contieneLeche(I)))).
✅ Parametrizado y expresivo
puedeComer(analia, Comida) :-
    comida(Comida),
    forall(ingrediente(Comida,I),
           aptoParaVegano(I)).

aptoParaVegano(I) :-
    not((indeseadoVegano(E),
         contiene(I, E))).

indeseadoVegano(carne).
indeseadoVegano(huevo).
indeseadoVegano(leche).

Al agregar un nuevo elemento indeseable para veganos (ej: miel), solo se agrega un hecho. El predicado puedeComer/2 no cambia.

Resumen Conceptos clave — Módulos 5 y 6

Módulo 5 · Recursividad

  • Un predicado recursivo se define en términos de sí mismo.
  • Necesita caso base (corte) y caso recursivo.
  • Permite encontrar múltiples soluciones a diferentes niveles de profundidad.
  • member(X, [X|_]). + member(X, [_|Z]):-member(X,Z). — inversible en primer argumento.
  • nth1/3 — posición N (desde 1) en una lista; inversible en todos sus argumentos.
  • La lista vacía no tiene caso porque viola el principio de universo cerrado.

Módulo 6 · Diseño

  • Diseño = componentes + responsabilidades + relaciones.
  • En Prolog: componentes son predicados e individuos (átomos, functores, números).
  • Functor ≠ predicado: un functor es una estructura de datos; un predicado define relaciones.
  • Acoplamiento necesario = dos componentes se conocen para resolver un requerimiento.
  • Buen acoplamiento: cambiar la implementación de un predicado no obliga a cambiar sus clientes.
  • 10 code smells: negación, duplicidad, over-engineering con listas, predicados vacíos, falta de inversibilidad, incógnitas, expresividad, hardcoding.
  • El antídoto general: pensar declarativamente, no algorítmicamente.
