Módulo 1

01Introducción al Paradigma Lógico

Hasta ahora conocimos distintas maneras de construir software. El paradigma lógico es una visión declarativa: en lugar de describir cómo resolver un problema paso a paso, describimos qué sabemos sobre el problema, y delegamos la estrategia de resolución a un motor externo.

Visión Imperativa / Procedimental

  • Secuencia de pasos a ejecutar
  • Estados intermedios en variables de memoria
  • Respeta arquitectura Von Neumann
  • El programador define el cómo

Visión Declarativa / Lógica

  • Sin algoritmo explícito
  • Solo definiciones (el qué)
  • Motor de inferencia construye el algoritmo
  • Concentración en lo esencial del problema
Concepto clave: Aplicación en el Paradigma Lógico
Una aplicación en el paradigma lógico tiene dos partes:
(1) Base de conocimiento — archivo .pl donde se declaran hechos y reglas con cualquier editor de texto.
(2) Consultas — el motor de inferencia de Prolog saca conclusiones a partir de ese conocimiento.
Módulo 1

02Predicados e Individuos

La base de conocimiento se construye mediante predicados e individuos. Veamos el ejemplo fundamental:

% Base de conocimiento: comidas.pl
pastas(ravioles).
pastas(fideos).
Definición: Predicado
Un predicado representa una relación o propiedad dentro del universo de conocimiento. Se identifica por su nombre y su aridad (cantidad de argumentos). Se escribe como nombre/aridad. En el ejemplo: pastas/1.
Definición: Individuo (átomo)
Los individuos son los elementos que participan de las relaciones. Se escriben siempre en minúscula (o entre comillas si contienen espacios). Son los "datos" del universo. Ejemplos: ravioles, fideos, juan, socrates.
Tipo de predicado Aridad Descripción Ejemplo
Monádico 1 Expresa una característica o atributo de un individuo pastas/1, animal/1
Poliádico 2 o más Expresa una relación entre dos o más individuos come/2, progenitor/2
Convención sintáctica importante
En Prolog, la distinción mayúsculas/minúsculas es fundamental:
• ravioles (minúscula) → individuo / átomo (un valor concreto)
• Pasta (mayúscula) → variable / incógnita (lo veremos en el Módulo 2)
Módulo 1

03Consultas de Verificación

Una vez cargada la base de conocimiento, se abre una consola donde se disparan consultas. El motor de inferencia evalúa si la relación consultada puede probarse a partir del conocimiento disponible.

Las consultas de verificación tienen todos sus argumentos instanciados (son individuos concretos). El resultado es siempre true o false.

pastas(ravioles).
true

pastas(fideos).
true

pastas(asado).
false
⚠ El punto final es obligatorio
Toda consulta en Prolog debe terminar con un punto .. Si se omite, Prolog muestra el símbolo | esperando que se continúe la consulta en la siguiente línea. Sin el punto, la consulta no se evalúa.
Módulo 1

04Principio de Universo Cerrado

Ante una consulta que no puede probarse a partir de la base de conocimiento, Prolog tiene una respuesta definida. Existen dos filosofías posibles:

Universo Abierto

  • Tres estados: verdadero, falso y desconocido
  • Lo que no se puede probar es "desconocido"
  • Algunos lenguajes semánticos (OWL, RDF)

Universo Cerrado (Prolog)

  • Solo dos estados: verdadero y falso
  • Todo lo que no se puede probar es falso
  • Prolog, SQL y similares
Principio de Universo Cerrado (CWA)
En Prolog, todo aquello que no figure en la base de conocimientos —o que no pueda derivarse de ella— se asume falso. La base de conocimiento define el universo completo de lo que es cierto; lo demás no existe.
Consecuencia práctica clave
No tiene sentido escribir hechos negativos. Bajo el universo cerrado, si asado no aparece en las cláusulas de pastas/1, ya sabemos que pastas(asado) es falso. Escribir explícitamente "el asado no es una pasta" sería redundante e incorrecto en Prolog.

% CORRECTO: simplemente no se escribe
pastas(ravioles).
pastas(fideos).

% INCORRECTO / SIN SENTIDO: el universo cerrado ya lo asume falso
no_es_pasta(asado). ← innecesario
Definición: Hecho y Cláusula
Un hecho es una afirmación codificada en la base de conocimiento con la sintaxis predicado(arg1, arg2, ...)., que se asume verdadera por el solo hecho de estar escrita. Cada hecho es una cláusula. Más adelante, las reglas también son cláusulas.
Módulo 1

05Predicados Poliádicos

Cuando una relación involucra más de un individuo, se usan predicados con más de un argumento (aridad ≥ 2). Expresan relaciones entre individuos.

come(juan, ravioles).
come(brenda, fideos).
gusta(brenda, fideos).
% El orden de los argumentos importa
come(brenda, fideos).
true

come(fideos, brenda).
false
← invertir orden no es equivalente

% La aridad debe coincidir exactamente
come(brenda, fideos, ricos).
ERROR: Undefined procedure: come/3
← come está definido como come/2, no come/3
⚠ Errores comunes con predicados poliádicos
1. Orden de argumentos: las relaciones no son bidireccionales por defecto. come(juan, pizza) no implica come(pizza, juan). El orden lo define el programador y debe ser consistente.

2. Aridad incorrecta: consultar come/3 cuando solo existe come/2 produce un error de "procedimiento indefinido".

3. Intercalar cláusulas: es buena práctica escribir todas las cláusulas de un mismo predicado juntas (contiguas). Si se intercalan con otros predicados, Prolog emite una advertencia.
Módulo 1

06Definición por Extensión

Cuando definimos un predicado listando explícitamente todos los individuos que lo satisfacen, hablamos de una definición por extensión. Es equivalente a enumerar los elementos de un conjunto.

animal(tigre).
animal(oso).
animal(elefante).
% Esto es como decir: Animales = { tigre, oso, elefante }

planeta(venus).
planeta(tierra).
Desventajas de la definición por extensión
• Requiere enumerar todos los elementos del conjunto manualmente.
• Para conjuntos grandes es incómodo e impráctico.
• No aporta información sobre qué característica comparten los elementos.
• La alternativa (que veremos en el Módulo 2) es la definición por comprensión, usando reglas.
Módulo 2

07Variables y Consultas Existenciales

Hasta ahora preguntábamos si algo específico se cumplía. Las variables (incógnitas) permiten preguntar qué individuos satisfacen una relación. En Prolog, toda palabra que comienza con mayúscula es una variable.

% Base de conocimiento
pastas(ravioles).
pastas(fideos).

come(juan, ravioles).
come(melina, ravioles).
come(brenda, fideos).
come(juan, fideos).
% Consulta existencial: ¿qué individuos son pastas?
pastas(Pasta).
Pasta = ravioles ;
Pasta = fideos .

% ¿Quiénes comen ravioles?
come(Persona, ravioles).
Persona = juan ;
Persona = melina ;
false

% ¿Qué come juan?
come(juan, Comida).
Comida = ravioles ;
Comida = fideos ;
false
Navegación de resultados múltiples
Cuando hay más de una solución posible, Prolog muestra una a la vez y espera. Para ver la siguiente solución: presionar ; (punto y coma), n, espacio o tab. Para detener la búsqueda: presionar . o Enter.

Variable Anónima _

Cuando queremos preguntar si existe algún individuo que satisfaga una relación, sin importar cuál es, usamos la variable anónima _ (guión bajo).

% ¿Existe alguna pasta?
pastas(_).
true

% ¿Existe alguien que coma algo?
come(_, _).
true
Tipos de consulta — Resumen
Tipo Argumentos Ejemplo Resultado
Verificación Todos instanciados (individuos concretos) come(juan, ravioles). true / false
Existencial con variable anónima Algún argumento es _ pastas(_). true / false
Existencial con variable nombrada Algún argumento es una variable (mayúscula) pastas(Pasta). Los valores que satisfacen la relación

08Inversibilidad

Definición: Inversibilidad
Un predicado es inversible cuando admite consultas con variables libres en sus argumentos: es decir, cuando puede responderse tanto instanciando el argumento (verificación) como dejándolo libre (consulta existencial). Los hechos son totalmente inversibles porque no tienen restricciones.
% come/2 es inversible: funciona con cualquier combinación
come(juan, ravioles).
true
← ambos instanciados
come(Persona, ravioles).
Persona = juan ; Persona = melina
← primer argumento libre
come(juan, Comida).
Comida = ravioles ; Comida = fideos
← segundo argumento libre
come(Persona, Comida).
Persona = juan, Comida = ravioles ; ...
← ambos libres

No siempre todos los predicados son totalmente inversibles. Las reglas que usan operaciones aritméticas o ciertos predicados built-in pueden restringir qué argumentos pueden dejarse libres. Veremos esto más adelante en la materia.

Módulo 2

09Reglas (simples)

Además de los hechos (axiomas), Prolog permite escribir reglas: predicados cuya verdad se deriva de otros predicados. Las reglas modelan la implicación lógica.

Silogismo → Regla en Prolog
El razonamiento clásico de Sócrates se formaliza así:

∀(x)(humano(x) ⇒ mortal(x))  →  "todo humano es mortal"
humano(socrates)  →  "Sócrates es humano"

En lógica se escribe p ⇒ q. En sintaxis Prolog (cláusula de Horn) se escribe al revés: q :- p. 
% Regla: mortal si es humano (cláusula de Horn)
mortal(Persona) :- humano(Persona).

% Hecho: Sócrates es humano
humano(socrates).
humano(platon).
mortal(socrates).
true

mortal(deadpool).
false
← humano(deadpool) no puede probarse → mortal(deadpool) falla

mortal(Quien).
Quien = socrates ;
Quien = platon .
Estructura de una regla
consecuente :- antecedente1, antecedente2, ...

• Consecuente: el predicado que se intenta satisfacer (cabeza de la cláusula).
• Antecedentes: condiciones que deben cumplirse para que el consecuente sea verdadero.
• Si todos los antecedentes se satisfacen, el consecuente también.
• Un predicado puede ser hecho (sin antecedentes) o regla (con al menos uno).
Ventaja declarativa
No tiene sentido escribir mortal(socrates). como hecho adicional, porque esa información ya puede derivarse de la regla y del hecho humano(socrates). Agregar el hecho sería redundante. Esto es una de las ventajas del paradigma: evitar repetición de información.
Módulo 2

10Unificación vs. Asignación

Uno de los conceptos más importantes y diferenciadores del paradigma lógico es la distinción entre unificación y asignación.

Asignación (imperativo)

  • Guarda un valor en una variable de memoria
  • La variable puede cambiar de valor
  • x = x + 1 tiene sentido: incrementa x
  • Operación destructiva sobre la memoria

Unificación (lógico)

  • Resuelve una variable como incógnita
  • La variable se liga a un valor, no se "reasigna"
  • X = X + 1 es una contradicción
  • Es una condición, no una instrucción
📌 Trampa de examen frecuente
X = X + 1 en Prolog NUNCA se puede cumplir. No es un incremento: es una condición lógica que dice "X es igual a X más 1", lo cual es una contradicción. Si X vale 2, no puede valer 3 al mismo tiempo.

Además, los predicados en Prolog no devuelven valores. Un predicado se satisface o no. No puede usarse como expresión (como en un paradigma funcional o imperativo). Solo se pueden construir más reglas para unificar valores.

Backtracking

Cuando hay múltiples soluciones posibles, Prolog las encuentra mediante un mecanismo llamado backtracking: prueba cada camino posible, y si uno falla, retrocede y prueba el siguiente. Es el motor que permite que una consulta con variables devuelva múltiples resultados.

11Reglas Compuestas (AND / OR)

Una regla puede tener múltiples antecedentes conectados por operadores lógicos. Prolog usa una sintaxis específica para cada uno.

% Base de conocimiento
viveEn(tefi, lanus).
viveEn(gise, lanus).
viveEn(alf, lanus).
viveEn(dodain, liniers).

docente(alf).
docente(tefi).
docente(gise).
docente(dodain).

Conjunción — AND (coma)

"Cualquier docente que vive en Lanús es un afortunado": se deben cumplir ambas condiciones. En Prolog, la conjunción se representa con una coma ,.

% p ∧ q => r
% afortunado si (es docente) Y (vive en Lanús)
afortunado(Persona) :- docente(Persona), viveEn(Persona, lanus).
afortunado(alf).
true
← docente(alf) ✓ y viveEn(alf, lanus) ✓
afortunado(dodain).
false
← docente(dodain) ✓ pero viveEn(dodain, lanus) ✗
afortunado(Quien).
Quien = alf ;
Quien = tefi ;
Quien = gise .

Disyunción — OR (múltiples cláusulas)

"Si una persona es docente o vive en Lanús, es afortunada": se cumple con alguna de las condiciones. En Prolog, la disyunción se expresa repitiendo la cabeza de la cláusula en reglas separadas.

% p ∨ q => r
% Se satisface por cualquiera de las dos ramas
afortunado(Persona) :- docente(Persona).
afortunado(Persona) :- viveEn(Persona, lanus).
📌 Trampa de examen frecuente
Para expresar OR en Prolog, NO se usa coma ni ningún operador especial dentro de una sola regla (eso sería AND). Se deben escribir dos cláusulas separadas con la misma cabeza.

❌ INCORRECTO para OR:
afortunado(P) :- docente(P), viveEn(P, lanus). ← esto es AND

✅ CORRECTO para OR:
afortunado(P) :- docente(P).
afortunado(P) :- viveEn(P, lanus).

Reglas compuestas con varios antecedentes y variables

Las variables dentro de una regla se unifican progresivamente: primero se unifica con el primer antecedente, y ese valor ya conocido se usa en los siguientes.

% Mario gusta de las chicas morochas con onda, y también de Luisa
gusta(mario, Chica) :- morocha(Chica), tieneOnda(Chica).
gusta(mario, luisa).

% Todos los que gustan de Ana Y de Luisa, gustan de Laura (AND)
gusta(Alguien, laura) :- gusta(Alguien, ana), gusta(Alguien, luisa).

% Todos los que gustan de Ana O de Luisa, gustan de Laura (OR)
gusta(Alguien, laura) :- gusta(Alguien, ana).
gusta(Alguien, laura) :- gusta(Alguien, luisa).
El lenguaje natural puede ser engañoso
En castellano, "Pedro gusta de Ana y de Nora" puede leerse como AND, pero semánticamente significa que ambas relaciones son verdaderas por separado. En Prolog eso se codifica como dos hechos (OR):
gusta(pedro, ana).
gusta(pedro, nora).

El AND se usa cuando se requiere que ambas condiciones se cumplan simultáneamente para el mismo individuo.
Módulo 2

12Definición por Comprensión

Mientras que la definición por extensión enumera los elementos uno a uno, la definición por comprensión describe los elementos por las características que los definen, mediante reglas.

Por Extensión

  • Enumera cada elemento explícitamente
  • Conjunto de hechos para el mismo predicado
  • Limitado para conjuntos grandes
  • animal(tigre). animal(oso).

Por Comprensión

  • Define qué característica debe cumplir un elemento
  • Una regla que generaliza el conjunto
  • Potente para conjuntos indefinidos o derivados
  • mortal(P) :- humano(P).
% Definición matemática: A = { a ∈ ℕ / 2 < a < 6 }
a(X) :- X > 2, X < 6.

% Definición matemática: B = { p ∈ ℕ / p < 10, siendo p par }
b(X) :- X < 10, par(X).

% Ejemplo del módulo: afortunado si docente y vive en Lanús
afortunado(Persona) :- docente(Persona), viveEn(Persona, lanus).

13Resumen y Errores Frecuentes

Conceptos clave de ambos módulos

Predicado
Relación o propiedad. Se identifica por nombre/aridad.
Individuo (átomo)
Empieza con minúscula. Es un valor concreto del universo.
Variable
Empieza con MAYÚSCULA. Es una incógnita a resolver.
Variable anónima
_ cuando el valor existe pero no nos importa cuál.
Hecho (cláusula)
Afirmación sin antecedentes. Se asume verdadera por estar en la base.
Regla
consecuente :- antecedentes. Se satisface si todos los antecedentes se cumplen.
Universo Cerrado
Todo lo que no puede probarse se asume FALSO. No hay hechos negativos.
Unificación
Ligar una variable a un valor como incógnita. Distinto de asignación.
Inversibilidad
Predicado inversible = admite consultas con variables libres en sus argumentos.
AND (conjunción)
Coma entre antecedentes: q :- p1, p2.
OR (disyunción)
Dos cláusulas con la misma cabeza: q :- p1. y q :- p2.
Def. por extensión
Enumerar todos los elementos con hechos.
Def. por comprensión
Definir el conjunto por características usando reglas.
Backtracking
Mecanismo del motor para encontrar todas las soluciones posibles.

Errores frecuentes de examen

📌 Checklist — antes del examen

✓ Mayúsculas/Minúsculas: juan = individuo, Juan = variable. Confundirlos da resultados incorrectos silenciosamente.

✓ Orden de argumentos: come(juan, pizza) ≠ come(pizza, juan). Las relaciones no son bidireccionales.

✓ OR ≠ AND: OR se escribe con dos cláusulas separadas, NO con coma. La coma es AND.

✓ X = X + 1 es una contradicción, no un incremento. En lógica, una variable no puede ser igual a sí misma más uno.

✓ No escribir hechos negativos: bajo universo cerrado, lo que no está en la base ya es falso. Escribir no_es_pasta(asado). es redundante e incorrecto conceptualmente.

✓ Aridad exacta: consultar come/3 cuando solo existe come/2 genera un error, no simplemente false.

✓ Los predicados no "devuelven" valores: se satisfacen o no. No son funciones; no pueden usarse en expresiones.
