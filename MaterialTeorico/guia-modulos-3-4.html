● Módulo 3 — Estructuras de datos

En Prolog toda la información que modelamos existe como individuos. Estos pueden ser simples (atómicos, indivisibles) o compuestos (agrupan varios individuos). Comprender esta distinción es la base para escribir predicados expresivos y correctos.

Individuos simples

Átomos que no se pueden descomponer en partes.

Átomos / identificadores

Identificadores que empiezan con minúscula: juan, ravioles, pilar. Se comparan con = y \=.

Números

Enteros y decimales como literales: 8, 3.14. Soportan operadores aritméticos y comparaciones de orden.

Strings

Cadenas con espacios entre comillas dobles: "Jorge Luis Borges". Útiles para representar nombres compuestos como un solo individuo.

Variables

Empiezan con mayúscula o _: Valor, X, _. No son individuos pero actúan como incógnitas en los predicados.

Aritmética: el operador is/2

En Prolog, = es unificación estructural, no evaluación numérica. La expresión 2 + 3 no se reduce automáticamente a 5: son estructuras distintas.

Consultas de ejemplo
% Unificación estructural (NO aritmética)
?- 5 = 2 + 3.    false   % el número 5 ≠ la expresión 2+3
?- 2 + 3 = 3 + 2.  false   % estructuras diferentes
?- 2 + 3 = 2 + 3.  true

% is/2 evalúa el lado derecho y unifica con el izquierdo
?- 4 is 2 * 2.      true
?- Z is 2 * 2.      Z = 4

% Variable sin ligar en el lado derecho → ERROR
?- 5 is N + 2.     ERROR: not sufficiently instantiated
⚠ Restricciones de inversibilidad: los operadores >, <, >=, =<, las operaciones aritméticas (+, -, *, /) y el operador is/2 requieren que sus argumentos estén instanciados. No se puede dejar una variable libre en la parte derecha de is/2.
Predicado siguiente/2
% INCORRECTO: Siguiente = Valor + 1 no evalúa
siguiente(Valor, Siguiente):- Siguiente = Valor + 1. % ✗

% CORRECTO: is/2 evalúa la expresión
siguiente(Valor, Siguiente):- Siguiente is Valor + 1. % ✓

?- siguiente(4, N).  N = 5
?- siguiente(2, 3). true
Pattern Matching

Búsqueda por patrones en el paradigma lógico.

El pattern matching en Prolog funciona de forma diferente al paradigma funcional (Haskell). En Haskell, una función devuelve la primera cláusula que coincide (unicidad). En Prolog, el motor prueba todas las cláusulas posibles y devuelve todos los individuos que satisfacen la relación.

Comparación funcional vs. lógico
-- Haskell: devuelve SOLO la primera coincidencia
valor 0 = 1
valor numero = numero

% Prolog: explora TODAS las cláusulas
valor(0, 1).
valor(Incognita, Incognita).

?- valor(0, Cual).
Cual = 1 ; % unifica con la primera cláusula
Cual = 0 ; % también unifica con la segunda (Incognita=0)
ℹ Prolog no tiene chequeo de tipos. Una variable sin ligar (Incognita) puede unificar con cualquier individuo: un número, un string, un átomo, una lista, un functor, etc.
Individuos compuestos — Listas

Serie ordenada de elementos que pueden repetirse.

Las listas son la primera estructura de datos compuesta de Prolog. Son recursivas: una lista es o bien la lista vacía [], o bien un elemento (cabeza) seguido de otra lista (cola).

NotaciónDescripciónEjemplo
[]Lista vacía, caso base.
[a]Lista con un solo elemento.[borges]
[a, b, c]Lista con múltiples elementos.[1, 4, 9, 10]
[H|T]Cabeza H y cola T. Patrón recursivo.[1|[4,9,10]]

Pattern matching con listas

PatrónCoincide conNo coincide con
[]Lista vacía solamente.Cualquier lista con elementos.
[E]Lista de exactamente un elemento.[] o [a,b].
[H|T]Lista con al menos un elemento.[].
Predicado head/2 con pattern matching
% El _ ignora la cola (no nos importa)
head([Cabeza | _], Cabeza).

?- head([1, 2, 3], X).  X = 1
?- head([borges], X).   X = borges

Predicados built-in para listas

length/2 — longitud

Relaciona una lista con su longitud. Es inversible: puede recibir la lista o el número como incógnita.

?- length([picasso, dali], N).    N = 2
?- length([picasso, dali], 2).    true
?- length(Lista, 3).              % genera listas de longitud 3

member/2 — pertenencia

Verifica si un elemento pertenece a una lista. Inversible en el primer argumento (permite enumerar elementos). Si la lista es incógnita, pierde sentido.

?- member(5, [5, 8]).          true
?- member(4, [5, 8]).          false
?- member(X, [5, 8]).           X = 5 ; X = 8

append/3 — concatenación

Relaciona dos listas con su concatenación. Es muy inversible: cualquiera de sus tres argumentos puede ser incógnita.

?- append([1,3], [2], R).              R = [1,3,2]
?- append([1,3], S, [1,3,borges]).    S = [borges]
?- append(X, Y, [1,2]).              % da 3 soluciones posibles

nth0/3 y nth1/3 — elemento por posición

nth0 usa índice base 0; nth1 usa índice base 1. Ambos son inversibles en la posición.

?- nth0(2, [borges,cortazar,bioy], E). E = bioy
?- nth1(2, [borges,cortazar,bioy], E). E = cortazar
?- nth0(P, [borges,cortazar,bioy], cortazar). P = 1

Otros predicados útiles

Functores

Individuos compuestos con nombre y aridad.

Un functor permite agrupar información heterogénea relacionada bajo un nombre. Se parece sintácticamente a un predicado, pero no tiene valor de verdad: no se puede consultar ?- fecha(22, 08, 1979)., sino que se usa como argumento dentro de un predicado.

Predicado

Define una relación. Tiene valor de verdad (true/false). Ejemplo: nacio(karla, ...)

Functor

Es un individuo compuesto. No tiene valor de verdad. Ejemplo: fecha(22, 08, 1979)

Uso de functores
% fecha/3 es un functor (individuo compuesto)
nacio(karla, fecha(22, 08, 1979)).
nacio(sergio, fecha(14, 10, 1986)).

% Pattern matching dentro del functor
?- nacio(Quien, fecha(_, _, 1986)).
Quien = sergio

?- nacio(_, fecha(_, _, Anio)).
Anio = 1979 ; Anio = 1986

% No se puede usar una variable como nombre de functor
?- nacio(_, Functor(_, _, Anio)). % ✗ ERROR

Ventajas de los functores: polimorfismo

Con functores, predicados distintos (natación, fútbol, rugby) pueden ser manejados por el mismo predicado gracias al pattern matching. Si se agrega un nuevo deporte, el predicado principal no necesita modificarse.

Ejemplo integrador — deportistas
practica(ana, natacion([pecho, crawl], 1200, 10)).
practica(deby, futbol(2, 15, 5)).
practica(zaffa, rugby(pilar, 0)).

% Cuántas medallas por pattern matching en el functor
cuantasMedallas(natacion(_, _, M), M).
cuantasMedallas(futbol(M, _, _), M).
cuantasMedallas(rugby(_, M), M).

medallas(Alguien, M):-
  practica(Alguien, Deporte),
  cuantasMedallas(Deporte, M).

% Extendible: agregar polo no cambia medallas/2
practica(gise, polo(8)).
cuantasMedallas(polo(_), 0).
● Módulo 4 — Predicados de orden superior

Los predicados de orden superior son predicados que reciben otros predicados como argumentos. Esto permite razonar sobre el universo de soluciones de una consulta: negarlo, cuantificarlo universalmente, o recolectarlo en una lista.

not/1 — negación lógica

Cuantificación existencial negativa: "no existe ningún x tal que…"

not(P) se satisface cuando el predicado P falla. Implementa la Hipótesis del Mundo Cerrado: si algo no se puede probar como verdadero, se asume falso.

Ejemplo básico — números yeta en la ruleta
juega(julia, 3).  juega(beto, 6).  juega(dodain, 5).

% Un número es yeta si nadie apuesta por él
yeta(Numero):- not(juega(_, Numero)).

?- yeta(3).   false  % julia juega el 3
?- yeta(25).  true   % nadie juega el 25
?- yeta(N).   false  % ← no puede enumerar lo que NO existe

Inversibilidad con not/1

El problema: not no puede generar el universo de individuos que no cumplen algo, porque Prolog no conoce todos los posibles individuos. La solución es ligar las variables antes del not, usando un predicado generador.

Con predicado generador → inversible
% between/3 genera los números del 1 al 36
numeroRuleta(N):- between(1, 36, N).

yeta(Numero):-
  numeroRuleta(Numero),    % liga Numero → generador
  not(juega(_, Numero)).

?- yeta(N).
N = 1 ; N = 2 ; N = 4 ; ... (excluye 3, 5 y 6)
⚠ Regla clave de inversibilidad: para que un predicado que usa not sea inversible, todas las variables dentro del not deben estar ligadas antes de que el motor llegue a él.

Aridad 1: negando conjunciones

not tiene aridad 1. Para negar dos condiciones a la vez, se usan paréntesis para mantener la aridad.

% ERROR: not/2 no existe
?- not(10 > 3, juega(dodain, 5)).  % ✗

% CORRECTO: paréntesis internos forman una conjunción
?- not((10 > 3, juega(dodain, 5))).  % ✓
forall/2 — cuantificación universal

"Para todo individuo que cumple P, también cumple Q."

forall(Condicion, Conclusion) se satisface si todos los individuos que satisfacen Condicion también satisfacen Conclusion. En lógica: ∀x / P(x) ⇒ Q(x).

Ejemplo — terminó el año académico
materia(pdp, 2).  materia(proba, 2).  materia(sintaxis, 2).
nota(nicolas, pdp, 10).  nota(nicolas, proba, 7).  nota(nicolas, sintaxis, 8).
nota(malena, pdp, 6).   nota(malena, proba, 2).

aprobo(A, M):- nota(A, M, N), N >= 6.

terminoAnio(Alumno, Anio):-
  forall(materia(Materia, Anio), aprobo(Alumno, Materia)).

?- terminoAnio(nicolas, 2). true
?- terminoAnio(malena, 2).  false

Inversibilidad con forall/2

El forall no liga sus variables: si Alumno llega como incógnita, el motor interpretará "¿todos los alumnos terminaron el año?", que no es lo que queremos. La solución es ligar con un predicado generador antes del forall.

Con generadores para ambos argumentos
% alumno/1 se deriva por comprensión de nota/3
alumno(A):- nota(A, _, _).
anio(A):- materia(_, A).

terminoAnio(Alumno, Anio):-
  alumno(Alumno),    % liga Alumno
  anio(Anio),         % liga Anio
  forall(materia(Materia, Anio), aprobo(Alumno, Materia)).

?- terminoAnio(Alumno, 2).  Alumno = nicolas ; ...
⚠ Trampa del vacuo verdadero: forall(P, Q) es verdadero cuando P es false para todos (no hay casos). Por eso ?- terminoAnio(nicolas, 5). → true si no hay materias en el año 5. Fijar el dominio con un generador de año evita este problema.

Del not al forall y viceversa

Las dos formas son lógicamente equivalentes, pero forall es más legible:

Con forallEquivalente con not
forall(p(X), q(X)) not((p(X), not(q(X))))
∀x / p(x) ⇒ q(x) ∄x / p(x) ∧ ¬q(x)

Lógica de conjuntos con forall

% A está incluido en B
incluido(A, B):- forall(member(X, A), member(X, B)).

% A es disjunto de B (no comparten elementos)
disjuntos(A, B):- forall(member(X, A), not(member(X, B))).
findall/3 — recolección de soluciones

Recolecta todas las soluciones de una consulta en una lista.

findall(X, Consulta, Lista) es el predicado que permite "saltar" de las respuestas individuales de Prolog a una lista con todos los resultados. Es la puerta de entrada para operaciones sobre conjuntos de soluciones.

Anatomía de findall/3

1er arg: el individuo (o expresión) que queremos recolectar.
2do arg: la consulta que genera los valores de ese individuo.
3er arg: la lista resultado.

Ejemplo — hijos de Homero
padre(homero, bart).
padre(homero, maggie).
padre(homero, lisa).
padre(juan, fede).

cantidadDeHijos(Padre, Cantidad):-
  padre(Padre, _),         % generador: liga Padre
  findall(Hijo, padre(Padre, Hijo), Hijos),
  length(Hijos, Cantidad).

?- cantidadDeHijos(homero, N).  N = 3
?- cantidadDeHijos(P, N).
P = homero, N = 3 ; P = juan, N = 1

Inversibilidad de findall/3

Si una variable libre en la consulta del findall no está ligada antes de llegar a él, el predicado ignorará esa distinción y recolectará todas las soluciones sin filtrar. El efecto es incorrecto.

Sin generador — resultado incorrecto
% Padre sin ligar: findall recolecta hijos de TODOS
cantidadDeHijos(Padre, N):-
  findall(Hijo, padre(Padre, Hijo), Hijos),
  length(Hijos, N).

?- cantidadDeHijos(P, N).  N = 4  % suma bart+maggie+lisa+fede
ℹ findall y el vacío: si la consulta no tiene soluciones, findall devuelve [] (lista vacía) en lugar de false. Esto implica que cantidadDeHijos(bart, N) daría N=0 si usamos un generador de personas, o false si el generador es solo "los que son padres".

Consultas compuestas dentro del findall

El segundo parámetro puede ser una conjunción de condiciones, envuelta entre paréntesis para preservar la aridad 3 de findall.

Múltiples condiciones en el 2do parámetro
% Cantidad de hijos varones
cuantosPibes(P, N):-
  persona(P),
  findall(Hijo, (padre(P, Hijo), varon(Hijo)), Varones),
  length(Varones, N).

Functores dentro del findall

Se puede usar un functor como primer argumento para recolectar pares o tuplas de datos. Esto es ideal para construir "tablas" o estructuras compuestas.

Tabla de posiciones (puntaje + equipo)
% puntos/2 es un functor, no un predicado
?- findall(puntos(Equipo, Pts),
            puntaje(Equipo, Pts),
            Tabla).
Tabla = [puntos(boca, 30), puntos(river, 28), ...]

Aplicaciones frecuentes de findall

Ejemplos combinados
% Suma de minutos jugados por un niño
minutosQueJugo(Nene, Total):-
  nene(Nene),
  findall(Min, jugoCon(Nene, _, Min), Lista),
  sum_list(Lista, Total).

% Intersección de dos conjuntos
interseccion(Xs, Ys, Zs):-
  findall(E, (member(E,Xs), member(E,Ys)), Zs).

% Coleccionista: todos sus items son valiosos
coleccionista(A):-
  tiene(A, _),                               % generador
  forall(tiene(A, Cosa), vale(Cosa)).
